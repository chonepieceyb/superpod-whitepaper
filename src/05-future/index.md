# 未来演进

前几章分析了帕累托前沿的定义、兑现与度量，并给出了当前阶段的参考设计。但帕累托前沿不是静态的——它的维度、形状和外推速度都在持续变化。未来 2–3 年是下一代算力基础设施定型的关键窗口期。对超节点而言，"未来演进"并不是追逐单点新技术，而是回答一个更工程化的问题：**哪些技术变量会改写帕累托前沿的维度与形状，以及会以什么速度改写？**

本章承接前一章的参考设计，但关注点不同：第四章回答"在当前前沿上选哪些点"，而本章回答"未来哪些变量会重塑前沿本身，以及会以什么节奏重塑"。

## 演进的两个核心命题

"未来演进"的核心命题可以用两个互锁的问题来界定：

**前沿外推的速度约束：跟上指数增长轨道**

NVIDIA 的代际路线图提供了一条清晰的基准线：从 Hopper 到 Blackwell 到 Rubin，系统级推理性能每代提升约 10×（硬件 ~2× × 软件 ~5×），训练效率的 GPU 数量需求每代缩减至约 1/4。这意味着任何超节点架构选择，如果不能在 2–3 年的窗口期内把帕累托前沿向外推一个数量级，就会被下一代平台直接淘汰。因此，未来演进的每一项技术讨论，都需要回答：**它能在多大程度上加速前沿外推？**

**前沿形状的结构性变化：适应新的权衡维度**

同时，帕累托前沿的坐标轴本身也在发生变化。当工作负载从稠密训练演进到推理-reasoning 混合、长上下文 MoE、多模态 Agent 时，"tokens/s/user vs tokens/s/GPU"不再是唯一重要的权衡面。新的关键维度正在浮现：

- **能效（tokens/s/MW）**：功耗密度逼近物理极限，供电与散热成为硬约束
- **上下文容量（KV cache $/token）**：长上下文推理将内存成本推到与算力同等重要的位置
- **可重构性**：动态拓扑（OCS）使得帕累托前沿从静态曲线变为随工作负载调度的动态曲面

## 三组矛盾与技术路径

这条主线可以拆成三组相互咬合的矛盾与解法：

- **互联的物理瓶颈**：当 SerDes 速率提升、链路预算趋紧时，铜互联在插损、功耗、可维护性上的压力会迅速放大。光互联与光交换的引入，本质上是在更大物理尺度上把"带宽与抖动"重新做可控化，同时为可重构拓扑提供物理抓手。从帕累托视角看，LPO → NPO → CPO 的演进不是简单的"一代比一代好"，而是在**功耗 vs 集成度 vs 可维护性 vs 成熟度**空间中的前沿外推与形状重塑。

- **封装与内存的密度瓶颈**：算力密度的提升越来越依赖 2.5D/3D 集成与堆叠内存（HBM3e → HBM4）。它们把互联距离压缩到封装内，却把供电、散热、良率与测试的复杂度推到台前。Chiplet + UCIe 的模块化路径，本质上是在**算力密度 vs 良率 vs 成本 vs 互联带宽**的帕累托空间中寻找工程可行的最优权衡。

- **模型形态的结构性变化**：低精度（NVFP4）、长上下文、稀疏/MoE、多模态，会把系统压力从"峰值算力"转移到"内存带宽/容量、通信小包与尾延迟、以及调度与资源隔离"。模型技术的演进不是独立话题，而是用来定义互联、封装与系统软件的约束条件——它决定了帕累托前沿上哪些区域是当下最需要优化的。

因此，本章采用"互联—拓扑—封装—Chiplet—模型"五条线并行展开：既讨论可选技术路径，也强调它们背后的工程约束与系统取舍。
