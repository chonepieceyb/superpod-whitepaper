### 1. 内存语义的实现与使用

- **互联层的内存语义**
    - Scale-Up 协议需暴露读写/原子/屏障等事务语义，并明确缓存一致性与完成通知模型（协议特性详见架构章节3.2节）。
    - 地址全局化：全局物理地址由 Fabric Manager 分配，页表 PTE 携带目标 GPU ID 供交换芯片路由（硬件路由机制详见架构章节4.1节）。
    - 目标是"通信即访存"：读/写/原子直接在 HBD 内完成，CPU 内核退出数据路径，只做控制平面。

- **GPU 的内存一致性**
    - GPU 内部一致性模型通过 NVLink/Fabric 扩展：UVA + 多级页表使远程显存透明可寻址（UVA 定义见架构章节1.1节）。
    - 原子/订购与屏障需在协议层保证可见性，避免跨设备的乱序与失效（协议一致性语义详见架构章节3.2节）。

- **Scale-Up 域的统一编址**
    - UVA/全局物理地址映射在 NVSwitch 域扩展为全局 PA，硬件完成地址翻译与路由，不再依赖 CPU 作为数据路径（硬件地址空间设计详见架构章节4.1节）。
    - IOMMU/权限：稀疏地址空间、分级访问控制与隔离域是共享内存安全性的前提。
    - 访问链路：驱动为远程 GPU 分配对等 ID → 页表把 VA 指向全局 PA → MMU 翻译得到目标 GPU → NVSwitch 路由到对端显存，完成如同本地的读写。

- **现代 GPU 访存体系（节点内）**
    - GPU 通过 UVA 将显存/主存映射到统一虚拟地址，页表支持多级映射与 4K–128K 页。
    - 节点内 P2P：驱动分配 Peer ID，页表标记 Peer，MMU 翻译后直接经 NVLink/PCIe 访问对端显存（NVLink 协议详见架构章节3.2节）。
    - 内核只在资源管理时介入，数据路径由硬件直接完成。

- **超节点访存体系（跨节点）**
    - 传统 PCIe P2P 受限于单一 Root Complex，无法跨物理节点统一访存（PCIe 架构详见架构章节4.1节）。
    - 超节点引入**全局物理地址空间**：Fabric Manager 划分 47-bit 全局 PA 段，PTE 标记目标 GPU（硬件地址空间设计详见架构章节4.1节）。
    - GPU 访问 VA → MMU 翻译出全局 PA + 目标 GPU → NVSwitch/Fabric 按地址路由到对端显存，CPU 不再参与数据面。
    - 价值：跨节点统一寻址 + 低延迟读写，为单边通信与共享内存编程奠定硬件基础。

### 2. 通信库的架构与设计

- **集合通信语义在超节点上的收益**
    - AllReduce / Broadcast / ReduceScatter / AllGather 在 HBD 域可近似线速，瓶颈在跨 HBD 收敛比与拥塞（HBD 定义见架构章节1.1节，拓扑设计详见架构章节3.3节）。
    - 分层通信：HBD 内高频小粒度，HBD 间聚合/稀疏化，提升全局吞吐。
    - 对比 BSP 模式：传统 SPMD + 屏障在非均匀负载（MoE、长序列）下放大 straggler 问题，超节点需要更灵活的通信语义与分层调度。

- **通信算法和拓扑**
    - **经典通信算法**：Ring、Tree、Mesh 等算法在不同拓扑下的性能特征（拓扑类型详见架构章节3.3节）。
    - **拓扑感知的算法选择**：根据物理拓扑（Fat-Tree、Dragonfly、NVSwitch Full-Mesh）选择最优通信算法（拓扑设计详见架构章节3.3节）。
    - **通信算法选择与分层算法设计**：HBD 内采用低延迟算法（如 Ring），HBD 间采用高带宽算法（如 Tree），实现分层优化。

- **All-to-All 性能分析**
    - 小包抖动与大包拥塞是 A2A 的主要瓶颈；路由感知/负载感知的调度可减少热点（路由机制详见架构章节3.3节）。
    - 稀疏 A2A（MoE/分段推理）需结合拓扑感知与动态路由（拓扑设计详见架构章节3.3节）。

- **PGAS 内存模型与单边通信**
    - Put/Get/原子等单边操作将"通信"退化为"内存访问"，天然适应非均匀负载（内存语义详见本章第1节）。
    - GPU 运行时协同：Pinned 内存、内存注册、RDMA doorbell 映射、完成通知队列等。
    - 混合编程故事线：在 MoE 场景，热专家与冷专家负载差异大——集合通信易被慢专家拖累；使用单边通信将结果直接写入对端内存，再由对端按需消费，降低全局同步成本。

- **SPMD/BSP vs PGAS/单边通信**
    - 通信模式对比：双边（Send/Recv 配对，强同步）vs 单边（Put/Get/原子，异步解耦）。
    - 性能风险对比：Straggler 放大 vs 异步/稀疏友好。
    - 适用场景对比：稠密训练、负载均匀 vs MoE、长序列推理、异步数据流。
    - 实现要点：集合通信库（NCCL/HCCL）vs NVSHMEM/PGAS 语义、完成通知、内存注册。
