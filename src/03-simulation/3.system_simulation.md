# 3. 负载驱动的系统仿真（System Simulation）：映射与编排

在全生命周期建模仿真体系中，**系统层（System Layer）**构成了连接上层逻辑需求与下层物理供给的核心推演引擎。继第二章确立了平台无关的负载规约，及第四章定义了物理资源的性能边界之后，本章聚焦于系统层的核心职能：**映射（Mapping）**与**编排（Scheduling）**。

系统层的主要任务是构建一个确定性的执行环境，将抽象的属性化有向无环图（Attributed DAG）投影至具体的物理集群拓扑中。这一过程包含两个阶段：
1.  **静态空间映射**（3.1节）：定义算子级粒度的物理部署位置，并将高层逻辑原语转化为硬件可执行的底层指令序列。
2.  **动态时序编排**（3.2节/3.3节）：在离散时间轴上模拟指令流的执行、竞争、同步与依赖关系，分别适配训练场景的同步批处理特征与推理场景的异步事件驱动特征。

## 3.1 映射空间与原语转换（Mapping & Lowering）

映射空间定义了从逻辑域（Logical Domain）到物理域（Physical Domain）的转换规则。该阶段作为仿真推演的前置静态配置环节，旨在完成计算图的**物理实例化（Physical Instantiation）**与通信原语的**底层展开（Primitive Expansion）**。

### 3.1.1 逻辑-物理映射（Logical-Physical Mapping）

本小节定义逻辑计算节点 $V_{logical}$ 与物理设备集合 $D_{physical}$ 之间的拓扑绑定关系，确立仿真的空间坐标系。

* **算子布局（Operator Placement）**
    建立逻辑设备 ID 到物理集群拓扑坐标的映射函数。仿真引擎维护全局**设备映射表（Device Map）**，将逻辑 `Rank` 映射为物理三元组 `(Rack_ID, Node_ID, Accelerator_ID)`。该映射关系直接决定了后续通信链路的物理属性（如 NVLink、PCIe 或 RDMA 网络）及相应的延迟与带宽约束。

* **并行策略实例化（Parallelism Instantiation）**
    依据用户定义的分布式并行策略（TP/PP/DP），对原始计算图执行**图变换（Graph Transformation）**，生成适配物理集群规模的执行图：
    * **张量切分（Tensor Tiling）**：针对张量并行（TP）场景，根据切分度数 $N$，将单体算子（如 MatMul）分解为 $N$ 个子核（Sub-kernels），并建立子核与物理设备的绑定关系。
    * **逻辑复制（Logical Replication）**：针对数据并行（DP）场景，在多个设备组间克隆模型参数与计算子图，构建并行执行副本。
    * **胶水节点插入（Glue Node Insertion）**：在切分后的子图边界自动生成必要的通信原语节点（如 `AllReduce`、`AllGather` 或 `P2P_Send/Recv`），以保证变换后的计算图在语义上的等价性。

### 3.1.2 通信原语降级（Communication Lowering）

负载模型中定义的通信语义（如“全局梯度聚合”）属于高层逻辑意图。系统层须依据物理网络拓扑与集合通信库（CCL）的具体实现，将高层意图**降级（Lowering）**为微观传输指令。

* **拓扑感知算法展开（Topology-aware Algorithm Expansion）**
    集合通信操作的性能表现取决于底层算法与物理拓扑的匹配度。系统层内置**集合通信算法库**，执行如下展开逻辑：
    * **算法选择**：依据物理连接关系（如 Ring, Mesh, Torus, Fat-tree），选择更适配的通信算法（例如：Ring-AllReduce、Tree-AllReduce 或 Hierarchical-AllReduce）。
    * **原语分解**：将集合操作分解为点对点（P2P）传输序列。例如，在 $N$ 节点的 Ring-AllReduce 算法中，将逻辑操作展开为 $2(N-1)$ 个 stage 的 `Send`/`Recv` 指令对；实际实现中可通过分块与流水化机制实现阶段级重叠，以刻画计算与通信的并发行为。

* **传输分块机制（Transmission Chunking）**
    为精确模拟大规模训练中的**计算-通信重叠（Communication-Computation Overlap）**特性，系统层引入数据分块策略。将大粒度传输张量切分为微块（Chunks），并在依赖图中建立细粒度的“块级依赖”。这使得仿真器能够捕捉到流水线执行中的微观并发行为，而非仅计算宏观总线带宽。

### 3.1.3 资源仲裁配置（Resource Arbitration Config）

物理环境中的计算单元、内存总线与网络接口均为竞争性资源。映射阶段需预定义资源争抢时的仲裁策略与流控规则。

* **流与队列映射（Stream & Queue Mapping）**
    定义逻辑执行流（Execution Stream）与硬件工作队列（Hardware Work Queues）的对应关系：
    * **通道隔离与并发**：配置计算流、H2D/D2H 拷贝流与 P2P 通信流在 DMA 引擎上的映射策略，确立不同类型指令在硬件层面的并发执行能力与互斥约束。
    * **优先级配置**：设定不同硬件队列的调度优先级（Priority Level），以模拟高优先级控制流对计算资源的抢占行为。

* **协议开销建模（Protocol Overhead Modeling）**
    除数据传输时延外，系统层引入对通信协议栈开销的量化建模：
    * **连接建立与同步延迟**：模拟跨节点通信的连接建立、控制面协商（如信用交换、队列初始化）等固定开销，以及逻辑屏障（Barrier）的同步等待时间。
    * **软件栈延迟**：量化内核态/用户态上下文切换、驱动层队列管理带来的固定指令周期开销（Latency Overhead）。

## 3.2 训练场景仿真：同步流水线编排

训练负载具有典型的**强同步、周期性、批处理（Synchronous, Periodic, Batch-oriented）**特征。其执行过程以迭代（Iteration/Step）为基本单位，在并行切分（TP/PP/DP）与一致性约束的共同作用下形成稳定的时序结构。相较于静态计算图分析，训练场景下的系统效率主要由同步传播、资源竞争与空转行为所决定，因此需引入面向时间轴的编排建模，用于刻画吞吐、利用率及长尾性能的形成机制。

### 3.2.1 训练执行流的时序结构：迭代系统与同步约束

训练执行流可视为受一致性约束驱动的迭代系统，其时序特征主要体现在：

* **迭代边界（Iteration Boundary）**
    每一轮训练存在清晰的逻辑边界，迭代边界与同步点共同决定训练的吞吐上界及稳态特征。

* **一致性约束（Consistency Constraint）**
    梯度一致性与参数一致性要求引入多级同步语义（如梯度聚合与更新边界），导致训练系统呈现“慢者决定整体”的全局耦合特征。

* **多尺度耦合（Multi-scale Coupling）**
    训练时序同时受算子依赖、跨 Stage 因果依赖、跨 Replica 同步依赖共同约束，形成多层嵌套的依赖结构与同步传播路径。

### 3.2.2 流水线编排的目标：将同步系统逼近稳态吞吐上界

在固定负载需求与物理资源约束下，训练编排的核心目标是最大化稳态有效工作时间，并压缩同步等待与空转开销。

* **稳态吞吐（Steady-state Throughput）**
    训练系统的主要优化对象是迭代吞吐而非单次延迟；吞吐由同步点、资源占用连续性与调度扰动共同决定。

* **有效工作占比（Effective Work Ratio）**
    流水线编排关注“设备在时间轴上的有效工作占比”，其损失主要表现为流水线空转与同步等待。

* **策略差异的统一刻画（Unified Characterization）**
    典型编排策略（如 1F1B、Interleaved、ZB1P）可在统一框架下比较：它们的本质差异体现为依赖结构松紧、粒度设置与稳态占用率的不同取舍。

### 3.2.3 关键现象：气泡、同步放大与重叠衰减

训练系统中可观测的性能损失通常表现为三类典型现象：

* **流水线气泡（Pipeline Bubble）**
    气泡体现为空转区间，可由结构性依赖、阶段负载不均衡与资源争抢等多种因素共同触发。

* **同步放大效应（Straggler Amplification）**
    在强同步体系下，局部拖尾会放大为全局迭代时间的增长，进而影响稳态吞吐与性能稳定性。

* **重叠有效性衰减（Overlap Degradation）**
    计算-通信重叠存在理论上限，但实际有效性将受到队列竞争、链路瓶颈与协议开销抑制，导致“可重叠”与“实际重叠”之间出现系统性差距。

### 3.2.4 训练显存行为：峰值、生命周期与可行性边界

训练显存不仅是静态预算，更是由生命周期与同步边界共同塑造的动态资源曲线。

* **峰值显存与出现位置（Peak Memory & Where It Happens）**
    激活值、梯度与优化器状态的生命周期交织决定峰值出现位置；峰值显存约束将直接影响并行策略可行空间与吞吐上界。

* **生命周期锁定效应（Lifecycle Lock-in）**
    同步点将部分存储对象在特定阶段“锁定”在显存中，形成训练场景特有的周期性压力模式。

* **策略代价曲线（Trade-off Curve）**
    重计算、卸载等策略可在显存压力与算力/带宽压力之间交换，其价值体现在对吞吐上界与峰值约束的重新塑形。

### 3.2.5 评价维度：吞吐、效率与可解释性

训练编排的评价应同时覆盖性能结果与系统行为两类维度：

* **吞吐与效率（Throughput & Efficiency）**
    - Step Time / Throughput
    - Pipeline Efficiency（有效工作时间占比）
    - Synchronization Penalty（同步等待占比）

* **行为可解释性（Interpretability）**
    - Bubble 分解：结构性 / 负载不均衡 / 资源竞争
    - 重叠有效性：理论上限 vs 实际达成
    - 拖尾诊断：关键路径与同步传播

* **可扩展性与稳定性（Scalability & Stability）**
    - 并行度扩展下的效率变化趋势
    - 对系统扰动（网络抖动、算子波动）的敏感性
    - 异常工况下的退化模式
